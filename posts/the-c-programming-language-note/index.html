C 程序设计语言——又称作「 K&R 」——是 C 语言祖师爷写的一本薄薄的书，里面很清晰透彻的讲解了 C 语言的语法和一些库函数、数据结构的设计思想，是一本入门和深入理解 C 都很合适的黑书。

这里记录一下自己的读书笔记，因为这本书大多内容都很基础，所以这里着重记录了一些我自己注意到的细节。我个人认为最重要的 第八章 会详细的记录一下。

如果要完整地学习，强烈建议 大家自己从头到尾将书阅读一遍。本书的少数语法已经有了更改（比如书中的 main 函数都没有声明返回值）或已经被淘汰，但是对整体来说影响并不大。

第一章

  • 在兼有值与赋值两种功能的表达式中，赋值结合次序是 从右至左 。即 nl = nw = nc = 0 等价于 nl = (nw = (nc = 0)) 。

第二章

  • 枚举类型的优势：相对于 #define 语句来说，它的优势在于常量值可以 自动生成 。并且枚举变量提供 合法检查（即存储的值是否为该枚举的有效值）。
  • 存储在字符变量中的位模式在某些机器中可能是 负 的，而在另一些及其可能是正的。为了保证程序的可移植性，最好给 char 指定 signed 或 unsigned 。（我个人觉得现在的大部分的机器都应该是整数。）
  • 表达式中 float 类型的数 不会 自动转换为 double 类型。
  • 当把 double 类型转换为 float 类型时，是进行四舍五入还是截取取决于具体的实现。
  • 当对 signed 类型的带符号值进行右移时，某些机器将对左边空出的部分用符号位填补（即「算数移位」），而另一些机器则用 0 填补（即「算数移位」）。
  • C 语言没有指定同一运算符中多个操作数的计算顺序（ && , || , ?: , , 除外），例如 x = f() + g() 中 f() 和 g() 的运算顺序 不定 。所以应该注意 printf("%d %d\n", ++n, power(2, n))（ ++n 可能在 power 之前或之后调用，得到不同的结果），a[i] = i++ 这种可能会出现不同结果的语句。

第三章

  • goto 语句可以运用在某些深度嵌套的结构中，用于跳出多层循环；如果需要在许多位置跳向同一个位置，goto 语句也能派上用场。但在大多数情况下 goto 使程序难以理解并且降低效率，所以还是尽量少地使用它。

第四章

  • 函数的声明和定义必须 一致 。如果函数单独编译（即函数定义放在另一个文件里），那么编译器 无法检测 出这种错误，返回的结果就可能毫无意义。
  • 如果函数带有参数，则要声明它们；如果没有参数，则使用 void 声明。
  • 如果要在外部变量的定义之 前 使用该变量，或者外部变量的定义与变量的使用 不在 同一个源文件中，则 必须 在相应的变量声明中强制性地使用关键字 extern 。
  • 外部变量的定义中必须指定数组的长度，但 extern 声明则不一定要指定数组的长度。
  • 使用 static 声明变量与函数可以达到隐藏外部对象的目的。（也就是说用了 static 指定为静态存储之后除了对声明的文件可见之外，其他文件都无法访问。）
  • #if 语句中的常量整型表达式 不包含 sizeof ，类型转换符或 enum 常量。

第五章

  • 只要保证元素存在，p[-1] 之类的表达式在语法上都是 合法 的。

  • 在指针 p 做加法 p + n 时，n 将根据 p 指向的对象的长度 按比例 缩放。（ p 指向的对象的长度取决于 p 的声明。）

  • 合法的指针运算：

    1. 相同类型 指针之间的赋值运算；
    2. 指针同整数之间的加法或减法运算；
    3. 指向 相同数组中元素 的两个指针间的减法或比较运算。

  • 字符串常量占据的存储单元数比双引号内字符数大 1（因为有 '\0' ）。

  • 字符串常量赋值没有涉及字符串的复制，只是涉及到 指针操作 。如果使用这个指针进行修改操作（也就是试图修改字符串常量的值），结果是 未定义 的。

第六章

  • *p->str++ 先读取指针 str 指向的对象的值，然后再将 str 的值加 1 ；(*p->str)++ 将指针 str 指向的对象的值加 1 ；*p++->str 先读取指针 str 指向的对象的值，然后将 p 加 1 。
  • 数组末尾之后的第一个元素（即 &tab[n] ）的指针算术运算可以正确执行。
  • 结构的长度 不等于 各成员长度之和。（因为不同的对象有不同的对齐要求，所以结构中可能会出现未命名的「 hole 」）。应使用 sizeof 来返回正确的对象长度。
  • 联合的目的：一个变量可以合法地保存多种数据类型中任何一种类型的对象。联合对象必须足够大以保存所有类型中最大的一种。而确定联合究竟使用的哪一种类型 需要程序员自己跟踪 。
  • 联合只能用其 第一个 成员类型的值进行初始化。
  • 位字段：在 结构体 里声明的特殊类型。声明格式为 type [name]: width 。width 定义存储单元个数（同具体实现有关，一般就是一位）。如 unsigned int foo: 1 声明一个一位字段，unsigned int 保证其为无符号量。若没有 name ，此 无名字段 起填充作用（在结构体里）。特殊宽度 0 用来强制对齐。
  • 位字段的所有属性几乎都同具体的实现有关。某些机器上字段的分配是从左到右，有些则相反。
  • 位字段操作执行速度并不高。

第七章

  • 变长参数表：在 <stdarg.h> 中定义。va_list 类型声明一个变量，该变量将依次引用各参数（比如声明一个变量 ap ，称为「参数指针」）。宏 va_start 将 ap 初始化为指向第一个无名参数的指针。（在使用 ap 之前必须调用一次，参数表至少包括一个有名参数，且 va_start 将最后一个有名参数作为起点。）

  • 通过调用 va_arg ，函数返回下一个参数，并将 ap 移向下一个参数。va_arg 还要传入一个类型名来决定返回的对象类型、指针的移动步长。

  • 最后必须调用 va_end 来完成一些必要的清理工作。

  • 启动一个 C 语言程序的时候，操作系统环境负责打开 3 个文件，分别是标准输入、标准输出和标准错误，对应的指针分别为 stdin , stdout , stderr 。大多数情况中，stdin 指向键盘，stdout 和 stderr 指向显示器。

  • 在 scanf 格式串中遇到一个或多个连续的空白符时，scanf 将从输入中 重复读空白字符直到遇到一个非空白字符 。

第八章

这一章主要讲 UNIX 系统中用于输入和输出的系统调用，并介绍如何通过它们实现标准库，内容硬核且底层。

文件描述符

UNIX 系统中，所有的外围设备（包括键盘、显示器）都被看做 文件 ，所有的输入 / 出都要通过读写文件完成。这样通过 单一的接口 就可以处理外围设备和程序之间的所有通信。

读写文件通常分三个步骤：

  1. 打开文件：将这个意图通知系统；
  2. 系统检查你的权力，若一切正常，操作系统将向程序返回一个小的非负整数，称为 文件描述符 ；
  3. 以 文件描述符 为标识（而不是以文件名为标识）进行输入 / 出。

系统负责维护已打开文件的所有信息，用户只能通过文件描述符引用文件。

之前提到过 shell 运行程序会打开三个文件——标准输入、标准输出和标准错误，其文件描述符分别为 0, 1, 2 。

fopen 和 getc 函数的实现

标准库中的文件不是通过文件描述符而是 文件指针 描述的。文件指针包含（主要）：

  1. 一个指向缓冲区的指针，通过它能一次性读入文件一大块内容；
  2. 一个记录缓冲区中剩余字符数的计数器；
  3. 一个指向缓冲区中下一个字符的指针；
  4. 文件描述符；
  5. 描述读写模式的标志；
  6. 描述错误状态的标志。

#define NULL 0
#define EOF (-1)
#define BUFSIZ 1024
#define OPEN_MAX 20 /* max #files open at once */

typedef struct _iobuf {
  int cnt;    /* characters left */
  char *ptr;  /* next character position */
  char *base; /* location of buffer */
  int flag;   /* mode of file access */
  int fd;     /* file descriptor */
}FILE;
extern FILE _iob[OPEN_MAX];

#define stdin (&_iob[0])
#define stdout (&_iob[1])
#define stderr (&_iob[2])

enum _flags {
  _READ = 01,  /* file open for reading */
  _WRITE = 02, /* file open for writing */
  _UNBUF = 04, /* file is unbuffered */
  _EOF = 010,  /* EOF has occurred on this file */
  _ERR = 020   /* error occurred on this file */
};

int _fillbuf(FILE *);
int _flushbuf(int, FILE *);

#define feof(p) ((p)->flag & _EOF) != 0)
#define ferror(p) ((p)->flag & _ERR) != 0)
#define fileno(p) ((p)->fd)
#define getc(p) (--(p)->cnt >= 0 ? (unsigned char)*(p)->ptr++ : _fillbuf(p))
#define putc(x, p) (--(p)->cnt >= 0 ? *(p)->ptr++ = (x) : _flushbuf((x), p))
// gethcar和putchar并不是函数而是宏
#define getchar() getc(stdin)
#define putcher(x) putc((x), stdout)

宏 getc 如果将计数器变为了负值，便调用 _fillbuf 填充缓冲区，重新初始化结构的内容，并返回一个字符。同样的，putc 将缓冲区填满的时候，它将调用函数 _flushbuf 。

#include "syscalls.h"
#include <fcntl.h>
#define PERMS 0666 /* RW for owner, group, others */
FILE *fopen(char *name, char *mode) {
  int fd;
  FILE *fp;
  if (*mode != 'r' && *mode != 'w' && *mode != 'a')
    return NULL;
  for (fp = _iob; fp < _iob + OPEN_MAX; fp++)
    if ((fp->flag & (_READ | _WRITE)) == 0) break; /* found free slot */
  if (fp >= _iob + OPEN_MAX) /* no free slots */
    return NULL;
  if (*mode == 'w') fd = creat(name, PERMS);
  else if (*mode == 'a') {
    if ((fd = open(name, O_WRONLY, 0)) == -1) fd = creat(name, PERMS);
    lseek(fd, 0L, 2);
  } else fd = open(name, O_RDONLY, 0);
  if (fd == -1) /* couldn't access name */
    return NULL;
  fp->fd = fd;
  fp->cnt = 0;
  fp->base = NULL;
  fp->flag = (*mode == 'r') ? _READ : _WRITE;
  return fp;
}

"syscalls.h" 和 <fcntl.h> 在第八章前面部分有说明，定义了一些宏（如 O_WRONLY ）以及一些函数（如 lseek 随机访问文件位置）。我们可以看出 fopen 的功能是打开文件，定位到合适位置，然后设置标志位以指示相应的状态。

#include "syscalls.h"
/* _fillbuf: allocate and fill input buffer */
int _fillbuf(FILE *fp) {
  int bufsize;
  if ((fp->flag & (_READ | _EOF_ERR)) != _READ)
    return EOF;
  bufsize = (fp->flag & _UNBUF) ? 1 : BUFSIZ;
  if (fp->base == NULL) /* no buffer yet */
    if ((fp->base = (char *)malloc(bufsize)) == NULL)
      return EOF; /* can't get buffer */
  fp->ptr = fp->base;
  fp->cnt = read(fp->fd, fp->ptr, bufsize);
  if (--fp->cnt < 0) {
    if (fp->cnt == -1) fp->flag |= _EOF;
    else fp->flag |= _ERR;
    fp->cnt = 0;
    return EOF;
  }
  return (unsigned char)*fp->ptr++;
}

#include "syscalls.h"

int _flushbuf(int x, FILE *fp) {
  unsigned nc;
  int bufsize;
  if (fp < _iob || fp >= _iob * OPEN_MAX) return EOF;
  if ((fp->flag & (_WRITE | _ERR)) != _WRITE) return EOF;
  bufsize = (fp->flag & _UNBUF) ? 1 : BUFSIZ;
  if (fp->base == NULL) {/* no buffer yet */
    if ((fp->base = (char *)malloc(bufsize)) == NULL) { /* can't get buffer */
      fp->flag |= _ERR;
      return EOF; 
    }
  } else { /* buffer already exists */
    nc = fp->ptr - fp->base;
    if (write(fp->fd, fp->base, nc) != nc) {
      fp->flag |= _ERR;
      return EOF; 
    }
  }
  fp->ptr = fp->base;
  *fp->ptr++ = (unsigned char) x;
  fp->cnt = bufsize - 1; 
  return x;
}

int fclose(FILE *fp) {
  int rc;
  if ((rc = fflush(fp) != EOF)) {
    free(fp->base); /* free allocated buffer */
    fp->ptr = NULL;
    fp->cnt = 0;
    fp->base = NULL;
    fp->flag &= ~(_READ | _WRITE);
  }
  return rc;
}

int fflush(FILE *fp) {
  int rc = 0;
  if (fp < _iob || fp >= _iob * OPEN_MAX) return EOF;
  if (fp->flag & _WRITE) rc = _flushbuf(0, fp);
  fp->ptr = fp->base;
  fp->cnt = (fp->flag & _UNBUF) ? 1 : BUFSIZ;
  return rc;
}

fclose 将对文件指针进行重置，保证之后 fopen 调用所分配到的指针不会有无意义的值。操作成功 fclose 将返回 0 。

#include "syscalls.h"

int fseek(FILE *fp, long offset, int origin) {
  unsigned nc;
  long rc = 0;
  if (fp->flag & _READ) {
    if (origin == 1) offset -= fp->cnt; /* remember chars in buffer */
    rc = lseek(fp->fd, offset, origin);
    fp->cnt = 0; /* no chars bufferd */
  } else if (fp->flag & _WRITE) {
    if ((nc = fp->ptr - fp->base) > 0)
      if (write(fp->fd, fp->base, nc) != nc) rc = -1;
    if (rc != -1) rc = lseek(fp->fd, offset, origin);
  }
  return (rc == -1) ? -1 : 0;
}

存储分配程序

malloc 需要 实时 申请空间，并且其管理的空间不一定是连续的。这样，空闲存储空间以 块循环链表 的方式 按存储地址升序 组织，每一个块包含一个长度、指向下一个块的指针以及一个指向自身储存空间的指针。

当申请空间的时候，malloc 扫描链表直到找到一个足够大的块位置。如果块恰好与请求的大小相符合，则将其移出链表并返回给用户；如果太大则分出一块合适的返回给用户；如果找不到一个足够大的块，则向操作系统申请一个大块加入链表中。

释放空间的时候，首先遍历链表找到合适的位置插入。如果有相邻的空闲块则将它们合并，以保证存储空间不会有太多碎片。

为了确保 malloc 返回的储存空间满足存储对象的对齐要求，所有块的大小必须是头部（块开始处的控制信息）大小的整数倍，且头部以正确地对齐。

typedef long Align; /* for alignment to long boundary */
typedef union header {      /* block header */
  struct{
    union header *ptr; /* next block if on free list */
    unsigned size;     /* size of this block */
  }s;
  Align x; /* force alignment of blocks */
}Header;

从代码中可以看出这是通过一个联合来实现的，其中假设最受限的类型为 long 。Align 永远不会被使用 ，它仅仅用于强制每个头部在最坏情况下仍满足对齐要求。

malloc 请求的长度将被舍入以保证是头部大小的整数倍。实际分配的块将多包含一个单元用于头部，而返回的的指针将指向空闲空间而不是头部。

static Header base; /* empty list to get started */
static Header *freep = NULL; /* start of free list */

void *malloc(unsigned nbytes) {
  Header *p, *prevp;
  Header *moreroce(unsigned);
  unsigned nunits;
  nunits = (nbytes + sizeof(Header) - 1) / sizeof(header) + 1;
  if ((prevp = freep) == NULL) { /* no free list yet */
    base.s.ptr = freeptr = prevptr = &base;
    base.s.size = 0;
  }
  for (p = prevp->s.ptr; ; prevp = p, p = p->s.ptr) {
    if (p->s.size >= nunits) { /* big enough */
      if (p->s.size == nunits) /* exactly */
        prevp->s.ptr = p->s.ptr;
      else { /* allocate tail end */
        p->s.size -= nunits;
        p += p->s.size;
        p->s.size = nunits;
      }
      freep = prevp;
      return (void *)(p + 1);
    }
    if (p == freep) /* wrapped around free list */
      if ((p = morecore(nunits)) == NULL)
        return NULL; /* none left */
  }
}

base 表示空闲块链表的头部，freep 表示上次找到空闲块的地方。初始时 freep 为 NULL ，系统将创建一个退化的链表，它只包含一个指向自己的大小为 0 的块。

每次请求空间的时候搜索都从 freep 开始（一次保证链表均匀）。如果找到很大的块则将尾部返回给用户，这样初始块头部就只用修改 size 了。

#define NALLOC 1024 /* minimum #units to request */ 

static Header *morecore(unsigned nu) {
  char *cp, *sbrk(int);
  Header *up;
  if (nu < NALLOC) nu = NALLOC;
  cp = sbrk(nu * sizeof(Header));
  if (cp == (char *)-1) /* no space at all */
    return NULL;
  up = (Header *)cp;
  up->s.size = nu;
  free((void *)(up + 1));
  return freep;
}

void free(void *ap) {
  Header *bp, *p;
  bp = (Header *)ap - 1; /* point to  block header */
  for (p = freep; !(bp > p && bp < p->s.ptr); p = p->s.ptr)
    if (p >= p->s.ptr && (bp > p || bp < p->s.ptr)) 
      break; /* freed block at start or end of arena */
  if (bp + bp->size == p->s.ptr) { /* join to upper nbr */
    bp->s.size += p->s.ptr->s.size;
    bp->s.ptr = p->s.ptr->s.ptr;
  } else bp->s.ptr = p->s.ptr;
  if (p + p->size == bp) { /* join to lower nbr */
    p->s.size += bp->s.size;
    p->s.ptr = bp->s.ptr;
  } else p->s.ptr = bp;
  freep = p;
}

UNIX 系统调用 sbrk(n) 返回一个指向 n 个字节储存空间的指针，若没有则返回 -1 。
